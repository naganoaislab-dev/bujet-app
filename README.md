# Budget Minus

予算から「あといくら使えるか」を考える、iPhone向けのオフライン家計簿PWAです。HTML、CSS、Vanilla JavaScriptだけで動作し、データはIndexedDBを使って端末内に保存します。

## 現在の機能

- 変動支出／固定支出を分けた予算カード
- カードから開く四則演算対応の金額入力電卓
- 支出・収入実績へのメモ追加
- 入力済み実績の金額、日付、種別、メモの編集と削除
- 種別ごとの今月残予算、これまでの持ち越し、予算消化率
- 支出種別ごとの日毎予算管理と、締日までの残日数に応じた今日の残予算
- 締日設定（短い月では月末へ自動調整）
- 管理する開始日／終了日の設定（初期値は36ヶ月）
- 変動支出、固定支出、収入の種別追加・編集・非表示
- 毎月同額／Nヶ月間隔の一括設定と、項目別の棒上限額・上限連動の操作単位による36ヶ月の月別調整
- 直近に編集した月から次月への計画値コピー
- 収入／支出の計画・実績、月次収支、累積収支のグラフと数値表
- 支出／収入を切り替えられる種別別の計画差、着地予測、未記録項目などの分析
- Excelで開けるUTF-8 CSVのエクスポート／インポート
- 完全復元用JSONバックアップ／復元
- オンライン／オフライン状態表示
- 初回読み込み後のオフライン起動
- iPhoneのセーフエリア、ホーム画面追加、ダークモード対応
- GitHub Pagesのサブディレクトリ配信対応
- 左上のアプリアイコンからのバージョン表示

## 画面

- **入力**: 起動時の画面。予算種別カードから支出を入力し、収入実績も記録できます。
- **状況**: 全管理期間の予定／実績と、月次／累積収支を確認できます。
- **分析**: 支出／収入を切り替え、月ごとの種別内訳、計画差、着地予測、注意点を確認できます。
- **設定**: 締日、期間、支出種別、収入種別、36ヶ月計画を変更できます。
- **データ**: CSV／JSONの入出力と初期化を行います。

## ローカル確認

Service Workerは `file://` では動作しません。任意の静的HTTPサーバーでリポジトリのルートを配信してください。

```sh
python -m http.server 8000
```

`http://localhost:8000/` を開き、次を確認します。

1. 変動支出または固定支出のカードを押す
2. 電卓で金額を入力して `OK` を押す
3. メモを入力して保存、または「メモなしで保存」を押す
4. 最近の記録を押し、内容を編集する
5. 設定画面で項目ごとの棒上限額を変更し、表示される操作単位で月別の棒を上下にドラッグして計画を調整する
6. 選択中の月を「次の月へコピー」し、選択表示も次月へ移ることを確認する
7. 支出種別の「日毎に予算管理」を切り替え、当月の入力カード表示が変わることを確認する
8. 左上のアプリアイコンを押し、バージョンを確認する
9. 状況／分析画面へ反映されることを確認する
10. 一度読み込みを完了してからオフラインへ切り替え、再読み込みする

## 締日の扱い

締日が25日の場合、「2026年7月分」は2026年6月26日〜7月25日です。締日が29〜31日でその日が存在しない月は、月末を締日として扱います。

種別ごとの残予算は次の式で毎回計算し、過去の実績を編集した場合も以降の持ち越しへ自動反映します。日毎管理を有効にした当月の種別は、それぞれの残予算を締日までの残日数（当日を含む）で割って「今日の残り予算」を表示します。

```text
今月の残り予算 = 当月計画 - 当月実績
持ち越し考慮後の残予算 = 当月計画 + 前月までの持ち越し - 当月実績
```

## Excel互換CSV

データ画面から、設定・種別・月別計画・実績を1つのUTF-8 CSVへ出力できます。Excelで開いて確認でき、本アプリが出力した列構成のままであれば再インポートできます。

インポートは現在のデータを全置換します。編集前にはJSON完全バックアップも保存してください。種別名やメモが `=`, `+`, `-`, `@` で始まる場合は、表計算ソフトの数式として評価されないよう出力時に無害化します。

## GitHub Pages

すべてのファイル参照、manifestの `start_url` / `scope`、Service Worker登録とキャッシュURLを相対指定しています。そのため、`https://<user>.github.io/<repository>/` のようなサブパスで動作します。

HTTPSで初回アクセス後、Safariの共有メニューから「ホーム画面に追加」を選択できます。

## 更新時の注意

- App Shellのファイルを変更したら、`index.html`・`manifest.webmanifest`・`app.js`・`sw.js` の資産URLに付けた `?v=...` と、`sw.js` の `CACHE_VERSION` を同じ世代へ更新してください。旧Service Workerの制御下でも新旧ファイルが混在しないために必要です。
- IndexedDBの構造を変更するときは `DB_VERSION` を増やし、既存データを失わない移行処理を追加してください。
- 金額は円単位の整数、業務日付は端末ローカルの `YYYY-MM-DD` として保存します。
- 重要なデータは定期的にJSONバックアップしてください。

## 既知の制約

- オフライン利用前に、オンライン状態で一度最後まで読み込み、Service Workerのインストールを完了する必要があります。
- SafariのプライベートブラウズやOSのストレージ整理により、IndexedDBのデータが削除される場合があります。
- CSVはExcelで開ける形式ですが、複数シートを持つ厳密な `.xlsx` 形式ではありません。
- ホーム画面アイコンはSVGです。iOSバージョンによって表示が異なる場合は、180×180 PNG版apple-touch-iconを追加してください。
- Safariタブ版とホーム画面版など、同じ端末で複数画面を同時に開いて編集すると、最後に保存した画面の状態が優先されます。編集は1画面ずつ行ってください。
